# 手机扫码编程

## 工作原理

Glasshouse 在本机启动 HTTP 服务后，会生成一个**局域网地址**（如 `http://192.168.1.100:7008`）。手机扫描二维码后，通过同一 WiFi 网络访问该地址，即可在手机上操作 Claude Code。

## 为什么扫码后无法连接？

最常见的原因：

1. **不在同一局域网** — 手机和电脑必须连接到同一个 WiFi 网络（同路由器/同网络名）
2. **防火墙拦截** — macOS/Windows 防火墙可能阻止了入站连接，需要允许 Glasshouse 的端口通过
3. **公司/学校网络隔离** — 部分企业网络会隔离设备间的直接通信（AP 隔离）
4. **VPN 干扰** — 如果电脑或手机开启了 VPN，可能导致网络路径不通

## 网络安全提示

> ⚠️ Glasshouse 的局域网服务默认对同一网络内的所有设备可访问。

- 在**公共 WiFi**（咖啡厅、机场等）下使用时请注意安全，同一网络的其他人理论上可以访问你的服务
- Glasshouse 使用 **Token 验证**保护局域网访问（URL 中自动附带 token），未携带正确 token 的请求会被拒绝
- 建议在可信的家庭或办公网络中使用

## 突破局域网限制

如果你需要在不同网络环境下（如外出时）远程访问电脑上的 Claude Code，可以通过以下方式：

- **内网穿透工具** — 如 frp、ngrok、Tailscale 等，将本地服务暴露到公网
- **Glasshouse 插件** — 支持通过插件机制配置中间代理，实现跨网络访问（详见插件文档）
