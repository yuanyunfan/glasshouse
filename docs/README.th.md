# Glasshouse

ชุดเครื่องมือ Vibe Coding ที่กลั่นกรองมาจากประสบการณ์การพัฒนาภาคปฏิบัติ สร้างขึ้นบน Claude Code:

1. ยกระดับขีดความสามารถ — รัน /ultraPlan และ /ultraReview บนเครื่องท้องถิ่น เพื่อที่โค้ดโปรเจกต์ของคุณจะไม่ต้องถูกเปิดเผยทั้งหมดให้กับคลาวด์ของ Claude;
2. รองรับหลายอุปกรณ์ — เขียนโค้ดจากอุปกรณ์มือถือผ่านเครือข่ายท้องถิ่น เวอร์ชันเว็บปรับตัวเข้ากับทุกสถานการณ์เพื่อฝังในส่วนขยายเบราว์เซอร์หรือมุมมองแบ่งหน้าจอของระบบปฏิบัติการ และยังมีตัวติดตั้งแบบเนทีฟให้ใช้งานด้วย;
3. บันทึกครบถ้วน — ดักจับและวิเคราะห์เพย์โหลดของ Claude Code ได้อย่างสมบูรณ์ เหมาะอย่างยิ่งสำหรับการบันทึกล็อก การดีบัก การเรียนรู้ และการวิศวกรรมย้อนกลับ;
4. แบ่งปันความรู้ — มาพร้อมกับบันทึกการศึกษาและประสบการณ์ภาคปฏิบัติที่สะสมไว้ (มองหาไอคอน "?" ทั่วทั้งแอป);
5. คงประสบการณ์เนทีฟไว้ — เพียงเพิ่มขีดความสามารถของ Claude Code โดยไม่มีการเปลี่ยนแปลงใดๆ ที่เป็นแก่นสำคัญ คงประสบการณ์เนทีฟไว้ครบถ้วน;
6. รองรับโมเดลของบุคคลที่สาม — ใช้งานได้กับ deepseek-v4-*, GLM 5.1, Kimi K2.6 พร้อมความสามารถ cc-switch ในตัวสำหรับการสลับเครื่องมือของบุคคลที่สามแบบ hot-swap ได้ทุกเมื่อ

## วิธีใช้งาน

### ข้อกำหนดเบื้องต้น

- ตรวจสอบให้แน่ใจว่าได้ติดตั้ง Node.js 22.0.0+ แล้ว; [ดาวน์โหลดและติดตั้ง](https://nodejs.org)
- ตรวจสอบให้แน่ใจว่าได้ติดตั้ง Claude Code แล้ว; [คู่มือการติดตั้ง](https://github.com/anthropics/claude-code)

### ติดตั้ง ccv

#### ติดตั้งผ่าน npm

```bash
npm install -g @yuanyunfan/glasshouse --registry=https://registry.npmjs.org
```

#### ติดตั้งผ่าน Homebrew (แนะนำสำหรับ macOS / Linux)

```bash
brew tap yuanyunfan/glasshouse
brew install glasshouse
brew upgrade glasshouse   # สำหรับอัปเดต — ห้ามใช้ npm install -g สำหรับการติดตั้ง brew
```

### การเปิดใช้งาน

ccv เป็นตัวแทนแบบ drop-in ของ claude — อาร์กิวเมนต์ทั้งหมดจะถูกส่งผ่านไปยัง claude พร้อมกับเปิดใช้งาน Web Viewer

```bash
ccv                    # == claude (interactive mode)
```

คำสั่งที่ผู้เขียนใช้บ่อยที่สุดคือ:
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
                       # ccv ส่งผ่านอาร์กิวเมนต์การเปิดใช้งาน Claude Code ทั้งหมด — รวมเข้าด้วยกันได้ตามต้องการ
```

หลังจากเปิดใช้งานในโหมดเขียนโปรแกรม หน้าเว็บจะเปิดขึ้นโดยอัตโนมัติ

Glasshouse ยังมาในรูปแบบแอปเดสก์ท็อปแบบเนทีฟด้วย: [หน้าดาวน์โหลด](https://github.com/yuanyunfan/glasshouse/releases)


### โหมด Logger

หากคุณยังคงชื่นชอบเครื่องมือ claude แบบเนทีฟ หรือส่วนขยาย VS Code ให้ใช้โหมดนี้

ในโหมดนี้ การเปิด `claude` จะเริ่มกระบวนการบันทึกล็อกโดยอัตโนมัติ ซึ่งจะบันทึกล็อกของคำขอไปยัง ~/.claude/cc-viewer/*yourproject*/date.jsonl

เปิดใช้งานโหมด logger:
```bash
ccv -logger
```

เมื่อคอนโซลไม่สามารถพิมพ์พอร์ตเฉพาะได้ พอร์ตแรกตามค่าเริ่มต้นคือ 127.0.0.1:7008 หลายอินสแตนซ์ใช้พอร์ตเรียงลำดับเช่น 7009, 7010

ถอนการติดตั้งโหมด logger:
```bash
ccv --uninstall
```

### การแก้ไขปัญหา

หากคุณพบปัญหาในการเริ่มต้น Glasshouse นี่คือวิธีแก้ปัญหาขั้นสูงสุด:

ขั้นตอนที่ 1: เปิด Claude Code ในไดเรกทอรีใดก็ได้

ขั้นตอนที่ 2: สั่ง Claude Code ด้วยคำสั่งต่อไปนี้:

```
I have installed the Glasshouse npm package, but after running ccv it still doesn't work properly. Please check Glasshouse's cli.js and findcc.js, and adapt them to the local Claude Code deployment based on the specific environment. Keep the scope of changes as constrained as possible within findcc.js.
```

การปล่อยให้ Claude Code วินิจฉัยปัญหาด้วยตัวเองมีประสิทธิภาพมากกว่าการถามใครหรืออ่านเอกสารใดๆ!

หลังจากคำสั่งข้างต้นเสร็จสิ้น `findcc.js` จะถูกอัปเดต หากโปรเจกต์ของคุณต้องการการ deploy ท้องถิ่นบ่อยครั้ง หรือโค้ดที่ fork มาต้องแก้ไขปัญหาการติดตั้งบ่อยๆ การเก็บไฟล์นี้ไว้จะช่วยให้คุณคัดลอกได้อย่างง่ายดายในครั้งต่อไป ในขั้นตอนนี้ โปรเจกต์และบริษัทจำนวนมากที่ใช้ Claude Code ไม่ได้ deploy บน Mac แต่ใช้ในสภาพแวดล้อมที่โฮสต์บนเซิร์ฟเวอร์ ดังนั้นผู้เขียนจึงแยก `findcc.js` ออกมาเพื่อให้ติดตามการอัปเดตซอร์สโค้ดของ Glasshouse ได้ง่ายขึ้นในอนาคต


### คำสั่งอื่นๆ

ดูที่:

```bash
ccv -h
```

### โหมดเงียบ

โดยค่าเริ่มต้น `ccv` จะทำงานในโหมดเงียบเมื่อห่อหุ้ม `claude` ทำให้เอาต์พุตของเทอร์มินัลสะอาดและสอดคล้องกับประสบการณ์แบบเนทีฟ ล็อกทั้งหมดจะถูกจับในเบื้องหลังและดูได้ที่ `http://localhost:7008`

เมื่อกำหนดค่าเสร็จแล้ว ใช้คำสั่ง `claude` ตามปกติ เยี่ยมชม `http://localhost:7008` เพื่อเข้าถึงอินเทอร์เฟซการตรวจสอบ

### โหมด Logger (ดูเซสชัน Claude Code แบบสมบูรณ์)

<img width="1500" height="768" alt="image" src="https://github.com/user-attachments/assets/a8a9f3f7-d876-4f6b-a64d-f323a05c4d21" />


- จับคำขอ API ทั้งหมดจาก Claude Code แบบเรียลไทม์ รับประกันข้อความต้นฉบับ — ไม่ใช่ล็อกที่ถูกตัดทอน (สิ่งนี้สำคัญ!!!)
- ระบุและติดป้ายกำกับคำขอของ Main Agent และ Sub Agent โดยอัตโนมัติ (ประเภทย่อย: Plan, Search, Bash)
- คำขอ MainAgent รองรับ Body Diff JSON แสดงความแตกต่างแบบพับจากคำขอ MainAgent ก่อนหน้า (เฉพาะฟิลด์ที่เปลี่ยนแปลง/ใหม่เท่านั้น)
- คำขอแต่ละรายการแสดงสถิติการใช้งาน Token แบบ inline (input/output tokens, การสร้าง/อ่านแคช, อัตราการ hit)
- เข้ากันได้กับ Claude Code Router (CCR) และสถานการณ์พร็อกซีอื่นๆ — ย้อนกลับไปใช้การจับคู่รูปแบบเส้นทาง API

### โหมดการสนทนา

คลิกปุ่ม "โหมดการสนทนา" ที่มุมบนขวาเพื่อแยกประวัติการสนทนาทั้งหมดของ Main Agent เป็นอินเทอร์เฟซแชท:

<img width="1500" height="764" alt="image" src="https://github.com/user-attachments/assets/725b57c8-6128-4225-b157-7dba2738b1c6" />

- ยังไม่รองรับการแสดง Agent Team
- ข้อความของผู้ใช้จัดชิดขวา (ฟองสบู่สีน้ำเงิน) การตอบกลับของ Main Agent จัดชิดซ้าย (ฟองสบู่สีเข้ม)
- บล็อก `thinking` จะถูกพับโดยค่าเริ่มต้น เรนเดอร์เป็น Markdown — คลิกเพื่อขยายและดูกระบวนการคิด; รองรับการแปลด้วยคลิกเดียว (ฟีเจอร์ยังไม่เสถียร)
- ข้อความการเลือกของผู้ใช้ (AskUserQuestion) จะแสดงในรูปแบบ Q&A
- การซิงค์โหมดสองทาง: การเปลี่ยนเป็นโหมดการสนทนาจะเลื่อนอัตโนมัติไปยังการสนทนาที่สอดคล้องกับคำขอที่เลือก; การเปลี่ยนกลับเป็นโหมด raw จะเลื่อนอัตโนมัติไปยังคำขอที่เลือก
- แผงการตั้งค่า: สลับสถานะการพับเริ่มต้นสำหรับผลลัพธ์เครื่องมือและบล็อก thinking
- การเรียกดูการสนทนาบนมือถือ: ในโหมด CLI บนมือถือ แตะปุ่ม "เรียกดูการสนทนา" ที่แถบบนเพื่อเลื่อนมุมมองการสนทนาแบบอ่านอย่างเดียวออกมา เพื่อเรียกดูประวัติการสนทนาทั้งหมดบนมือถือ
