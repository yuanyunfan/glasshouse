# Glasshouse

Набір інструментів Vibe Coding, виокремлений з практичного досвіду розробки, побудований на основі Claude Code:

1. Підніміть стелю можливостей — запускайте /ultraPlan та /ultraReview локально, щоб код вашого проекту ніколи не мав бути повністю розкритий у хмарі Claude;
2. Адаптація до різних пристроїв — програмуйте з мобільних пристроїв у локальній мережі, веб-збірка адаптується до будь-якого сценарію для вбудовування в розширення браузера або розділені екрани ОС, також надається нативний інсталятор;
3. Повний слід аудиту — повне перехоплення та аналіз payload Claude Code, ідеально підходить для логування, налагодження, навчання та реверс-інжинірингу;
4. Обмін знаннями — постачається з накопиченими навчальними нотатками та практичним досвідом (шукайте іконки «?» по всьому додатку);
5. Нативний досвід збережено — лише розширює можливості Claude Code без жодних суттєвих змін у його ядрі, зберігаючи нативний досвід незмінним;
6. Підтримка сторонніх моделей — працює з deepseek-v4-*, GLM 5.1, Kimi K2.6, з вбудованою функцією cc-switch для гарячої заміни сторонніх інструментів у будь-який момент.

## Використання

### Передумови

- Переконайтеся, що Node.js 22.0.0+ встановлено; [завантажити та встановити](https://nodejs.org)
- Переконайтеся, що Claude Code встановлено; [інструкція зі встановлення](https://github.com/anthropics/claude-code)

### Встановити ccv

#### Встановлення через npm

```bash
npm install -g @yuanyunfan/glasshouse --registry=https://registry.npmjs.org
```

#### Встановлення через Homebrew (рекомендовано для macOS / Linux)

```bash
brew tap yuanyunfan/glasshouse
brew install glasshouse
brew upgrade glasshouse   # для оновлень — НЕ використовуйте npm install -g для brew-встановлень
```

### Запуск

ccv — це пряма заміна claude — усі аргументи передаються до claude під час запуску Web Viewer.

```bash
ccv                    # == claude (interactive mode)
```

НАЙчастіше використовувана автором команда:
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
                       # ccv передає всі аргументи запуску Claude Code — комбінуйте їх як завгодно
```

Після запуску в режимі програмування веб-сторінка відкриється автоматично.

Glasshouse також постачається як нативний десктопний застосунок: [Сторінка завантаження](https://github.com/yuanyunfan/glasshouse/releases)


### Режим Logger

Якщо ви все ще віддаєте перевагу нативному інструменту claude або розширенню VS Code, використовуйте цей режим.

У цьому режимі запуск `claude` автоматично запустить процес логування, який записує логи запитів до ~/.claude/cc-viewer/*yourproject*/date.jsonl

Увімкнути режим logger:
```bash
ccv -logger
```

Коли консоль не може надрукувати конкретний порт, за замовчуванням використовується перший порт 127.0.0.1:7008. Кілька екземплярів використовують послідовні порти, такі як 7009, 7010.

Видалити режим logger:
```bash
ccv --uninstall
```

### Усунення несправностей

Якщо ви стикаєтеся з проблемами при запуску Glasshouse, ось остаточний підхід до усунення несправностей:

Крок 1: Відкрийте Claude Code у будь-якому каталозі.

Крок 2: Дайте Claude Code наступну інструкцію:

```
I have installed the Glasshouse npm package, but after running ccv it still doesn't work properly. Please check Glasshouse's cli.js and findcc.js, and adapt them to the local Claude Code deployment based on the specific environment. Keep the scope of changes as constrained as possible within findcc.js.
```

Дозволити Claude Code самому діагностувати проблему ефективніше, ніж запитувати когось або читати будь-яку документацію!

Після виконання наведеної вище інструкції `findcc.js` буде оновлено. Якщо ваш проект часто потребує локального розгортання, або якщо форкнутий код часто потребує вирішення проблем встановлення, збереження цього файлу дозволить вам просто скопіювати його наступного разу. На цьому етапі багато проектів та компаній, що використовують Claude Code, розгортаються не на Mac, а в хостованих серверних середовищах, тому автор виділив `findcc.js`, щоб у майбутньому було легше відстежувати оновлення вихідного коду Glasshouse.


### Інші команди

Див.:

```bash
ccv -h
```

### Тихий режим

За замовчуванням `ccv` працює в тихому режимі під час обгортання `claude`, зберігаючи ваш термінальний вивід чистим і відповідним нативному досвіду. Усі логи захоплюються у фоновому режимі і можуть бути переглянуті за адресою `http://localhost:7008`.

Після налаштування використовуйте команду `claude` як зазвичай. Відвідайте `http://localhost:7008`, щоб отримати доступ до інтерфейсу моніторингу.

### Режим Logger (перегляд повних сесій Claude Code)

<img width="1500" height="768" alt="image" src="https://github.com/user-attachments/assets/a8a9f3f7-d876-4f6b-a64d-f323a05c4d21" />


- Захоплює всі API-запити від Claude Code у реальному часі, забезпечуючи сирий текст — не редаговані логи (це важливо!!!)
- Автоматично ідентифікує та маркує запити Main Agent та Sub Agent (підтипи: Plan, Search, Bash)

- Сумісний з Claude Code Router (CCR) та іншими проксі-сценаріями — повертається до зіставлення шаблонів шляхів API

### Режим бесіди

Натисніть кнопку «Режим бесіди» у верхньому правому куті, щоб розібрати повну історію бесіди Main Agent у інтерфейс чату:

<img width="1500" height="764" alt="image" src="https://github.com/user-attachments/assets/725b57c8-6128-4225-b157-7dba2738b1c6" />

- Відображення Agent Team ще не підтримується
- Повідомлення користувача вирівняні по правому краю (сині бульбашки), відповіді Main Agent вирівняні по лівому краю (темні бульбашки)
- Блоки `thinking` за замовчуванням згорнуті, відображаються як Markdown — натисніть, щоб розгорнути і переглянути процес мислення; підтримується переклад одним натисканням (функція все ще нестабільна)
- Повідомлення вибору користувача (AskUserQuestion) відображаються у форматі Q&A
- Двонаправлена синхронізація режимів: перемикання в режим бесіди автоматично прокручує до бесіди, що відповідає вибраному запиту; перемикання назад у сирий режим автоматично прокручує до вибраного запиту
- Панель налаштувань: перемкнути стан згортання за замовчуванням для результатів інструментів і блоків thinking
- Мобільний перегляд бесіди: у мобільному режимі CLI торкніться кнопки «Перегляд бесіди» у верхній панелі, щоб висунути вигляд бесіди тільки для читання для перегляду повної історії бесіди на мобільному пристрої
