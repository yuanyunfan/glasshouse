# Glasshouse

Набор инструментов Vibe Coding, извлечённый из практического опыта разработки и построенный на основе Claude Code:

1. Более высокий потолок возможностей — запускайте /ultraPlan и /ultraReview локально, чтобы код вашего проекта не был полностью раскрыт в облаке Claude;
2. Адаптация к нескольким устройствам — позволяет программировать с мобильных устройств (в локальной сети), веб-версия адаптируется к различным сценариям, легко встраивается в расширения браузера или разделённые экраны ОС, также доступен нативный установщик;
3. Полное сохранение логов — полный перехват и анализ payload Claude Code, идеально подходит для логирования, отладки, обучения и реверс-инжиниринга;
4. Обмен опытом обучения — накоплено множество учебных материалов и опыта разработки (ищите иконки «?» по всему приложению);
5. Сохранение нативного опыта — лишь расширяет возможности Claude Code без существенных изменений ядра, сохраняя нативный опыт;
6. Поддержка сторонних моделей — совместим с deepseek-v4-*, GLM 5.1, Kimi K2.6, встроенная функциональность cc-switch позволяет переключать сторонние инструменты на лету.

## Использование

### Предварительные требования

- Убедитесь, что установлен Node.js 22.0.0+; [Скачать](https://nodejs.org)
- Убедитесь, что установлен Claude Code; [Руководство по установке](https://github.com/anthropics/claude-code)

### Установка ccv

#### Установка через npm

```bash
npm install -g @yuanyunfan/glasshouse --registry=https://registry.npmjs.org
```

#### Установка через Homebrew (рекомендуется для macOS / Linux)

```bash
brew tap yuanyunfan/glasshouse
brew install glasshouse
brew upgrade glasshouse   # для обновлений — НЕ используйте npm install -g для brew-установок
```

### Запуск

ccv — это прямая замена claude — все аргументы передаются в claude при запуске Web Viewer.

```bash
ccv                    # == claude (interactive mode)
```

НАИБОЛЕЕ часто используемая автором команда:
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
                       # ccv передаёт все аргументы запуска Claude Code — комбинируйте их как угодно
```

После запуска в режиме программирования веб-страница откроется автоматически.

Glasshouse также поставляется как нативное десктопное приложение: [Страница загрузки](https://github.com/yuanyunfan/glasshouse/releases)


### Режим Logger

Если вы по-прежнему предпочитаете нативный инструмент claude или расширение VS Code, используйте этот режим.

В этом режиме запуск `claude` автоматически запустит процесс логирования, который записывает логи запросов в ~/.claude/cc-viewer/*yourproject*/date.jsonl

Включить режим logger:
```bash
ccv -logger
```

Когда консоль не может напечатать конкретный порт, по умолчанию используется первый порт 127.0.0.1:7008. Несколько экземпляров используют последовательные порты, такие как 7009, 7010.

Удалить режим logger:
```bash
ccv --uninstall
```

### Устранение неполадок

Если вы столкнулись с проблемами при запуске Glasshouse, вот окончательный подход к устранению неполадок:

Шаг 1: Откройте Claude Code в любом каталоге.

Шаг 2: Дайте Claude Code следующую инструкцию:

```
I have installed the Glasshouse npm package, but after running ccv it still doesn't work properly. Please check Glasshouse's cli.js and findcc.js, and adapt them to the local Claude Code deployment based on the specific environment. Keep the scope of changes as constrained as possible within findcc.js.
```

Позволить Claude Code самому диагностировать проблему более эффективно, чем спрашивать кого-либо или читать какую-либо документацию!

После выполнения вышеуказанной инструкции `findcc.js` будет обновлён. Если ваш проект часто требует локального развёртывания или если разветвлённый код часто должен решать проблемы установки, сохранение этого файла позволит вам просто скопировать его в следующий раз. На этом этапе многие проекты и компании, использующие Claude Code, развёртываются не на Mac, а в хостируемых серверных средах, поэтому автор выделил `findcc.js`, чтобы в дальнейшем было легче отслеживать обновления исходного кода Glasshouse.

### Тихий режим

По умолчанию `ccv` работает в тихом режиме при оборачивании `claude`, сохраняя ваш терминальный вывод чистым и соответствующим нативному опыту. Все логи захватываются в фоновом режиме и могут быть просмотрены по адресу `http://localhost:7008`.

После настройки используйте команду `claude` как обычно. Посетите `http://localhost:7008` для доступа к интерфейсу мониторинга.

### Режим Logger (просмотр полных сессий Claude Code)

<img width="1500" height="768" alt="image" src="https://github.com/user-attachments/assets/a8a9f3f7-d876-4f6b-a64d-f323a05c4d21" />


- Захватывает все API-запросы от Claude Code в реальном времени, обеспечивая сырой текст — не редактированные логи (это важно!!!)
- Автоматически идентифицирует и маркирует запросы Main Agent и Sub Agent (подтипы: Plan, Search, Bash)
- Запросы MainAgent поддерживают Body Diff JSON, показывая свёрнутые различия от предыдущего запроса MainAgent (только изменённые/новые поля)
- Каждый запрос отображает встроенную статистику использования Token (входные/выходные токены, создание/чтение кэша, коэффициент попаданий)
- Совместим с Claude Code Router (CCR) и другими прокси-сценариями — возвращается к сопоставлению шаблонов путей API

### Режим беседы

Нажмите кнопку «Режим беседы» в правом верхнем углу, чтобы разобрать полную историю беседы Main Agent в интерфейс чата:

<img width="1500" height="764" alt="image" src="https://github.com/user-attachments/assets/725b57c8-6128-4225-b157-7dba2738b1c6" />

- Отображение Agent Team пока не поддерживается
- Сообщения пользователя выровнены по правому краю (синие пузыри), ответы Main Agent выровнены по левому краю (тёмные пузыри)
- Блоки `thinking` свёрнуты по умолчанию, отображаются как Markdown — нажмите, чтобы развернуть и просмотреть процесс мышления; поддерживается перевод одним нажатием (функция пока нестабильна)
- Сообщения выбора пользователя (AskUserQuestion) отображаются в формате Q&A
- Двунаправленная синхронизация режимов: переключение в режим беседы автоматически прокручивает к беседе, соответствующей выбранному запросу; переключение обратно в режим сырых данных автоматически прокручивает к выбранному запросу
- Панель настроек: переключить состояние свёртывания по умолчанию для результатов инструментов и блоков thinking
- Мобильный просмотр беседы: в мобильном режиме CLI коснитесь кнопки «Просмотр беседы» на верхней панели, чтобы выдвинуть вид беседы только для чтения для просмотра полной истории беседы на мобильном устройстве

### Управление логами

Через выпадающее меню Glasshouse в левом верхнем углу:

<img width="1500" height="760" alt="image" src="https://github.com/user-attachments/assets/33295e2b-f2e0-4968-a6f1-6f3d1404454e" />

**Сжатие логов**
Что касается логов, автор хочет уточнить, что официальные определения Anthropic не были изменены, что обеспечивает целостность логов. Однако, поскольку отдельные записи логов от модели 1M Opus могут стать чрезвычайно большими на более поздних этапах, благодаря определённым оптимизациям логов для MainAgent достигается как минимум 66% уменьшение размера без gzip. Метод разбора этих сжатых логов можно извлечь из текущего репозитория.
